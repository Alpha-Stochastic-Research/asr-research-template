# Contributing to Alpha Stochastic Research

Use Issues for proposals and Pull Requests for reviewable changes. Keep each change focused, document assumptions, add tests where relevant and never commit secrets or confidential data.

## Scientific contributions

Include the research question, assumptions, methodology, data provenance, validation method, limitations and reproduction instructions.

## Software contributions

Use a dedicated branch, run the repository checks, document public interfaces and add regression tests. Prefer small pull requests with clear evidence.

## Review

A contribution is not approved solely because automated checks pass. Scientific, security, legal and editorial review remain human responsibilities.
